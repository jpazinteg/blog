## MSMQ について

MSMQ とは、弊社が提供するオンプレミス製品のメッセージ キューイング サービスです。
- [Message Queuing (MSMQ)](https://docs.microsoft.com/en-us/previous-versions/windows/desktop/legacy/ms711472(v=vs.85))

> Message Queuing (MSMQ) technology enables applications running at different times to communicate across heterogeneous networks and systems that may be temporarily offline. Applications send messages to queues and read messages from queues. 

## 事象の概要

[KB5071544](https://support.microsoft.com/ja-jp/topic/2025-%E5%B9%B4-12-%E6%9C%88-9-%E6%97%A5-kb5071544-os-%E3%83%93%E3%83%AB%E3%83%89-17763-8146-630aa62e-f399-4e42-9f7a-2a4d38dd1210) にて以下のような問題が発生し、MSMQ メッセージを送ろうとするとエラーが発生する可能性がございます。以下一部引用です。

[Windows 10 Version 1809とWindows Server 2019 の既知の問題と通知 | Microsoft Learn](https://learn.microsoft.com/ja-jp/windows/release-health/status-windows-10-1809-and-windows-server-2019) 

> メッセージ キュー (MSMQ) が 2025 年 12 月の Windows セキュリティ更新プログラムで失敗する可能性がある

> - MSMQ キューが非アクティブになる
> - IIS サイトが "操作を実行するためのリソース不足" エラーで失敗する
> - アプリケーションがキューに書き込むことができない
> - メッセージ ファイルの作成時に "メッセージ ファイル 'C:\Windows\System32\msmq\storage*.mq' を作成できません" などのエラー
> - 十分なディスク領域とメモリが使用可能であるにもかかわらず、"ディスク領域またはメモリ不足" のような誤解を招くログ

## 事象の発生条件

本事象は、MSMQ のサービスが起動されていない場合は影響を受けません。
 
 MSMQ が有効化されているかは、以下のコマンドよりご確認いただけます。

```
Dism /online /Get-FeatureInfo /FeatureName:MSMQ
```

また、サービスが起動しているかは、以下のコマンドよりご確認いただけます。 

```
sc query MSMQ
```

有効化されていない、またはサービスが起動していない場合は本問題の影響を受けませんのでご安心いただけるかと存じます。
